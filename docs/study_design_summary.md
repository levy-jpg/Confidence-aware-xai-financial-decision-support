## Design

The study uses a supervised within-subject design. Each participant completes both explanation conditions: Static and Adaptive.

## Participants and Responses

Target sample:

- 10 participants
- 6 applicant cases per participant
- 60 total response rows
- 30 Static responses
- 30 Adaptive responses

Minimum acceptable sample:

- 6 participants
- 6 applicant cases per participant
- 36 total response rows
- 18 Static responses
- 18 Adaptive responses

## Counterbalanced Condition Allocation

Each participant completes Applicant A to Applicant F in order. The app assigns the explanation condition automatically from the participant ID.

Pattern A for odd participant IDs, such as P001, P003, P005:

- Applicant A: Static
- Applicant B: Adaptive
- Applicant C: Static
- Applicant D: Adaptive
- Applicant E: Static
- Applicant F: Adaptive

Pattern B for even participant IDs, such as P002, P004, P006:

- Applicant A: Adaptive
- Applicant B: Static
- Applicant C: Adaptive
- Applicant D: Static
- Applicant E: Adaptive
- Applicant F: Static

This balances the order effect so some participants begin with Static and others begin with Adaptive. Each applicant also appears under both conditions across the whole study.

## Participant Procedure

For each applicant case, the participant should:

1. Start with the next applicant case in order.
2. Review the applicant profile.
3. Make an initial Good Credit or Bad Credit judgement.
4. Rate their confidence.
5. Submit the initial judgement.
6. View the AI prediction.
7. Complete the review checks.
8. Generate and read the explanation.
9. Complete the evaluation ratings for trust, understanding, usefulness, and reliance.
10. Add an optional comment.
11. Submit the evaluation.
12. Use the continue button to move to the next applicant case.

## Neutral Participant Script

Participants should be told:

> This is a study evaluating an AI decision-support prototype. You will review six fictional applicant profiles from a benchmark credit dataset. For each case, please make your own judgement first, then view the AI prediction and explanation. The study is evaluating the system and explanation design, not your personal ability. Please answer honestly based on your understanding and experience of each case.

Participants should not be told that one explanation condition is expected to be better, improved, or longer because of their confidence.

## Data Quality Checks

After each participant:

- check that the participant has exactly six saved rows
- check the dashboard totals
- confirm that Static and Adaptive responses remain balanced

If an invalid row is recorded, use the analysis dashboard Raw Data tab to delete the individual row. The dashboard automatically creates a backup before deletion.
